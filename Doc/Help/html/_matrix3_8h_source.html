264  // The unguaranteed order of initialization of static variables across
265  // translation units can be a source of annoying bugs, so now the static
266  // special values (like Vector3::ZERO, Color3::WHITE, ...) are wrapped
267  // inside static functions that return references to them.
268  // These functions are intentionally not inlined, because:
269  // "You might be tempted to write [...] them as inline functions
270  // inside their respective header files, but this is something you
271  // must definitely not do. An inline function can be duplicated
272  // in every file in which it appears ?and this duplication
273  // includes the static object definition. Because inline functions
274  // automatically default to internal linkage, this would result in
275  // having multiple static objects across the various translation

277  // ensure that there is only one definition of each wrapping
